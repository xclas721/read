# SA 課程需求訪談練習整理

**訪談人員：** Diego  
**日期：** 2026-06-02  
**主題：** 客戶資訊工具 / AI Agent 需求訪談

## 一、需求背景

本次需求訪談主要針對「客戶資訊工具」或「AI Agent 可調用的客戶資訊系統」進行初步需求盤點。

系統目標是讓內部使用者可以透過 CLI 或 AI Agent 快速查詢、維護與使用客戶相關資訊，並支援後續維運、SRE、系統整合與權限控管需求。

## 二、主管訪談重點

### 1. 回傳格式與 CLI 使用方式

確認系統在透過 CLI 查詢或調用工具時，回傳格式應該採用哪一種標準。

需要確認：

- CLI 查詢結果要回傳 JSON、Table、純文字，還是多種格式都支援？
- 根據 CLI 使用情境，是否需要有不同的回傳結果？

### 2. 工具權限：更新與刪除

需要確認 AI Agent 或 CLI 工具是否需要具備「新增、更新、刪除」資料的權限。

主管方向：可以提供工具更新與刪除權限，但需要搭配權限控管。

建議需求：

- 查詢權限：一般使用者可查詢授權範圍內的客戶資訊。
- 新增權限：特定角色可新增客戶資料。
- 更新權限：特定角色可修改聯絡資訊。
- 刪除權限：需限制高權限角色操作。
- 重要資料變更需保留 audit log。
- 系統變動類資料需要 approve 權限。

### 3. 聯絡資訊定期更新

客戶聯絡資訊需要定期維護，避免資料過期或聯絡不到對應窗口。

建議需求：

- 系統需支援聯絡資訊定期更新。
- 可設定定期提醒資料 owner 檢查資料。
- 每筆資料需紀錄最後更新時間。
- 可標示資料狀態，例如：有效、待確認、已停用。
- 若超過一定期間未更新，可提示使用者資料可能過期。

## 三、客戶資訊資料範圍

系統需納入以下客戶資訊欄位：

### 1. 客戶基本資訊

- 客戶名稱
- 服務類型（ACS or 3DSS）標籤
- Issuer OID
- 地區 / 國家
- SLA 等級
- 客戶狀態，例如：導入中、已上線、暫停服務、終止服務

### 2. 聯絡人資訊

- 聯絡人姓名（必填）
- 聯絡人角色標籤（必填）
- Email（必填）
- Mobile
- 其他通訊軟體資訊
- 客戶 IP

## 四、SRE 使用情境訪談

需要針對 SRE 角色確認實際使用情境，確保系統可以支援日常維運與事件處理。

### 1. IP 白名單新增

SRE 可能需要快速新增客戶或系統的 IP 白名單設定。

### 2. 增加 issuer oid、地區

可協助 SRE 快速判斷是否為單一客戶問題或區域性問題。

## 五、系統變動與審核權限

針對會影響系統行為或客戶服務的資料異動，需要加入 approve 權限。

需要審核的變動類型：

- IP 白名單異動
- SLA 等級異動
- 主要聯絡人異動
- 其他系統變動類資料

## 六、初步需求整理

| 類別 | 需求項目 | 說明 |
|---|---|---|
| 查詢介面 | CLI | 使用者可透過 CLI 查詢客戶資訊 |
| 回傳格式 | 根據場景定義 | 根據 CLI 場景定義 |
| 權限控管 | 查詢 / 新增 / 更新 / 刪除 | 不同角色需有不同操作權限 |
| 資料維護 | 聯絡資訊定期更新 | 避免客戶聯絡資訊過期 |
| 客戶資料 | 服務、聯絡人、OID、地區、SLA | 建立完整客戶資料模型 |
| SRE 情境 | IP 白名單 | 支援維運與問題排查 |
| 審核流程 | System 變動 approve | 重要異動需經審核 |
| 交易資料 | issuer oid、地區 | 支援交易查詢與問題定位 |

## 七、結論

本次需求訪談初步確認，客戶資訊工具需要支援客戶基本資料、聯絡資訊、服務資訊、SRE 維運情境、權限控管與系統變更審核流程。

系統初期可先以查詢功能為主，支援 CLI 與 AI Agent 調用；後續再逐步加入新增、更新、刪除與 approve 流程。

資料設計上，應將 issuer oid、地區、SLA、主要聯絡人、服務類型納入核心欄位，以便支援維運與交易問題排查。
